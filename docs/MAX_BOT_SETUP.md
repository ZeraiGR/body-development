# Создание бота в MAX и подключение второго канала

MAX (мессенджер VK, 2025) — второй канал доставки курса после Telegram. Цель: фоллбек
(если в TG не прочитано за 2ч — сообщение уходит в MAX), ручное переключение канала
(` /channel `) и команда теории дня (` /theory N `) из любого мессенджера. Архитектура
кросс-платформы описана в задаче #5 и `VPS_MANAGEMENT.md`.

> Статус (2026-07): создание бота в MAX требует верифицированного профиля (юрлицо/ИП/
> **самозанятый**). Физлицо бота создать не может. Ниже — путь для самозанятого.

---

## Предусловие: статус «Самозанятый» (обязательно)

С августа 2025 публиковать ботов в MAX могут только верифицированные профили. Самозанятость —
самый лёгкий легальный путь:

1. Приложение **«Мой налог»** (iOS/Android) → регистрация (паспорт + ИНН + фото + регион).
2. Налоговая ставка: 4% (доход от физлиц) / 6% (от юрлиц). ~10 минут.
3. ⚠️ Это налоговый статус: если нет freelance-дохода — налог 0₽, статус просто висит.
   Снимается в любой момент.
4. Альтернатива — ИП (тяжелее, отчётность). Для личного бота хватает самозанятости.

---

## Шаг 1. Бизнес-профиль в MAX

1. Открыть **[business.max.ru](https://business.max.ru)** (с компьютера).
2. Авторизоваться по номеру телефона.
3. Тип профиля → **«Самозанятый»**.
4. Пройти **верификацию через Госуслуги** (привязка «Мой налог»/Госуслуги).

## Шаг 2. Создать канал (формальность для паблика бота)

MAX ожидает наличие канала у бизнеса. Создать минимальный публичный канал:
- Название, описание, аватар, контакты.
- 3–5 постов (кто ты, чем занимаешься — можно «по-личному»; требуется для модерации).

## Шаг 3. Создать бота и получить токен

1. Кабинет business.max.ru → раздел **«Чат-боты»** (или **«Боты»**) → **Создать**.
   - Альтернатива: в MAX найти бота **@MasterBot** → создать там → команда **«Получить токен»**.
2. Заполнить: название, описание, аватар (для личного — любое).
3. **Ник бота** генерируется автоматически (`idИНН_bot` или `se(orgid)_bot`), изменению не подлежит.
4. Настройки бота → **«Расширенные настройки» → «Настроить»** → скопировать **токен**
   (строка вида `AAHdqTcvCH1vGWJxfSeofSAs0K5PALDsaw`).

## Шаг 4. Модерация (подождать)

Бот проходит модерацию — статус виден рядом с названием. До модерации пользователи не могут
его запустить. Обычно от часов до пары дней.

## Шаг 5. Запустить бота у себя

После модерации открыть диплинк в MAX (аналог TG `/start`):
```
https://max.ru/<имя_бота>?start=hello
```
Событие `bot_started` инициализирует диалог. Без него бот не может писать первым.

## Шаг 6. Передать токен в проект (безопасно)

Токен — секрет, **не коммитить и не пересылать в чат**. Положить в `.env` на VPS:
```bash
ssh root@194.62.66.34
nano /root/body-development/.env
# добавить строку (вставить свой токен вместо ...):
#   MAX_BOT_TOKEN=...
# Ctrl+O, Enter, Ctrl+X
systemctl restart back-bot
```
После этого — сборка MAX-бэкенда (см. ниже «Что собирается после токена»).

## Честные нюансы

- **Цена:** для самозанятого публикация бота, по источникам (гайд июня 2026), бесплатна.
  Точную сумму/её отсутствие подтвердит сама регистрация на business.max.ru (если плата
  есть — покажут перед оплатой). Оформление самозанятости бесплатно.
- **Интерфейс** business.max.ru может чуть отличаться — искать раздел «Чат-боты»/«Боты»
  и кнопку «Получить токен».
- **Без домена** — старт на long-poll (для одного пользователя работает; официально
  dev-режим, но для личного бота норм). Webhook + домен добавляются позже для production.

## Техническая справка по MAX Bot API (для сборки)

| Параметр | Значение |
|---|---|
| Домен API | `https://platform-api2.max.ru` (старый `platform-api.max.ru` — дедлайн миграции **19 июля 2026**) |
| Авторизация | заголовок `Authorization: <token>` (без `Bearer`); в query токен больше не передавать |
| Rate limit | 30 запросов/сек |
| Отправка сообщения | `POST /messages?user_id=<int64>` (ЛС) или `?chat_id=<int64>` (чат/канал) |
| Лимит текста | до 4000 символов |
| Форматирование | поле `format`: `"markdown"` или `"html"` |
| Inline-кнопки | вложение `inline_keyboard`, до 210 кнопок; типы `link` (ссылка) и `callback` (колбэк) |
| Колбэк | событие `message_callback`, идентификатор в `callback.callback_id` |
| Ответ на колбэк | `POST /answers?callback_id=...` — поле `message` обновляет сообщение (можно убрать кнопки `attachments: []`), `notification` — toast |
| Редактирование | `PUT /messages?message_id=...` (только сообщения младше 24 ч) |
| События (минимум) | `bot_started`, `message_created`, `message_callback` |
| Получение обновлений | **webhook** (`POST /subscriptions`, HTTPS порт 443, доверенный УЦ/Минцифры, ответ 200 за 30с) **или** long-poll (`GET /updates`) — взаимоисключающи |
| Аналог Telegraph | отсутствует → статьи хостить на Telegraph, в MAX слать ссылку + inline-кнопку |
| Python SDK | официальный `maxapi`, либо тонкий адаптер на `aiohttp` (выбрано для проекта) |

### Ключевые риски
1. **Business mode + верификация** — главный барьер (см. предусловие).
2. `format: "markdown"` — набор тегов проверить на стенде (страница Formatting в доках не грузится).
3. `Keyboard.button.callback` — известный баг документации (GitHub issue #216); сверяться с реальным поведением API.
4. **24-часовое окно** редактирования — для «вечерний фидбэк по утренней кнопке» может не подойти (>24ч → новое сообщение).
5. Хардкодить в конфиге только `platform-api2.max.ru`.

## Что собирается после токена (задача #5)

1. `bot/notifiers/` — абстракция `notifier` с двумя бэкендами: `TelegramTransport` (aiogram) и `MaxTransport` (`aiohttp` + long-poll).
2. Идемпотентный `task_done(date, action)` — единый источник правды для «выполнено» (ключ `(date, action)`, повторные клики no-op).
3. Фоллбек-оркестратор: утро → TG; если за 2ч нет «прочитано» → отправка в MAX.
4. `msg_refs(date, action, platform, chat_id, message_id)` — для синхронизации кнопок: при завершении на одной платформе — `POST /answers`/`PUT /messages` (MAX) и `editMessageReplyMarkup` (TG) убирают/меняют кнопку на другой.
5. Команды в обоих мессенджерах: `/theory N` (алиас `/day N`) — открыть теорию дня; `/channel telegram|max|auto` — ручной переключатель канала (`preferred_channel` в `user_state`, приоритет над авто-фоллбеком).
6. Статьи — единый источник (Telegraph, ссылки уже в `telegraph_links`), на обеих платформах одинаковые.
