# Бот реабилитации спины

Личный Telegram-бот-ассистент по реабилитации спины (aiogram 3.x + APScheduler + SQLite).

> 📐 Подробная техническая документация (архитектура, модель данных, конечный автомат,
> модель безопасности, журнал решений, выводы) — в **[ARCHITECTURE.md](ARCHITECTURE.md)**.
> Здесь — краткий стартовый гайд.
Однопользовательский: общается **только** с владельцем (`ALLOWED_CHAT_ID`) и устойчив к
промпт-инъекциям (полностью шаблонный, без LLM — см. `bot/security.py`).

Программа описана тремя файлами в корне:
- **CLAUDE.md** — тон, ежедневная архитектура (утро/день/вечер), механика метрик.
- **YEARLY_PLAN.md** — годовая стратегия (4 фазы).
- **MONTH_1_PLAN.md** — план на месяц (4 недели с действиями).

Бот берёт **действия по неделе** из `MONTH_1_PLAN`, **научный факт по этапу дня** из
`CLAUDE.md §3`, а **теорию** — из 30 статей (`bot/content/articles.py`, заливаются на Telegraph).

## Что делает

- **Утро** (`08:30`, настраивается): вода + 3 вдоха, ссылка на статью дня, ключевой факт
  (🟢 обязательно / 🟡 опционально), утренние действия недели. Кнопка «✅ Сделал».
- **День** (случайный момент в окне `13:00–17:00`): напоминание о микро-паузе.
- **Вечер** (`20:30`): интеграция с отдыхом (Дота/сериал + аппликатор), вечерний ритуал,
  сбор метрик `pain` → `stiffness` → `habits` (кнопки 1–10), затем мотивационный фидбек
  по формуле §5 с научным фактом текущего этапа.
- **Воскресенье**: мини-отчёт за неделю (sparkline метрик, средние/мин/макс). На дне 30 —
  итог месяца и готовность к Фазе 2.
- **Пропуск дня**: бот не ругает, streak обнуляется и восстанавливается на следующий лог.
- **«Тяжело»**: кнопка/команда задерживают текущую неделю ещё на 2 дня.

## Установка

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

cp .env.example .env
# заполни .env:
#   BOT_TOKEN        — токен от @BotFather
#   ALLOWED_CHAT_ID  — твой id (узнать у @userinfobot)
```

## Заливка теории на Telegraph (один раз)

```bash
python3 scripts/upload_telegraph.py
```

Создаст 30 статей и сохранит ссылки в БД. Если `TELEGRAPH_TOKEN` пуст — создаст аккаунт
и напечатает токен (сохрани его в `.env`, чтобы повторные запуски не плодили страницы).
Без залитых ссылок бот тоже работает — в утреннем сообщении будет заглушка вместо ссылки.

## Запуск

```bash
python3 main.py
```

Бот крутится long-polling, пока работает процесс. Напиши ему `/start`.

## Команды

| Команда    | Действие                                              |
|------------|-------------------------------------------------------|
| `/status`  | день/неделя/стрик/пауза                               |
| `/today`   | показать утреннее сообщение текущего дня вручную      |
| `/evening` | запустить вечерний опрос вручную                      |
| `/report`  | недельный отчёт с графиком                            |
| `/pause`   | `/resume` — пауза/снятие рассылок                     |
| `/next`    | перескочить на следующий день                         |
| `/hard`    | задержаться на неделе ещё на 2 дня                    |
| `/reset`   | начать программу заново с дня 1                       |

## Структура

```
config.py              env: токены, chat_id, таймзона, время рассылок, путь к БД
main.py                точка входа (polling)
bot/
  security.py          single-user guard + устойчивость к инъекциям
  db.py                SQLite: user_state / daily_logs / telegraph_links
  planner.py           перевод дня, недели, стадии, streak, sparkline
  messages.py          шаблоны утро/день/вечер/фидбек/отчёт (формула §5)
  handlers.py          команды + колбэки вечернего флоу
  scheduler.py         APScheduler: утро/пинг/вечер/воскресный отчёт
  telegraph.py         клиент Telegraph API
  content/
    articles.py        30 статей теории
    program.py         недели MONTH_1_PLAN + научные этапы §3
scripts/
  upload_telegraph.py  разовая заливка 30 статей
```

## Настройка расписания

Все тайминги — в `.env` (локальное время в `TIMEZONE`, по умолчанию `Europe/Moscow`):
`MORNING_TIME`, `EVENING_TIME`, `DAY_PING_START`, `DAY_PING_END`.
